# Ubuntu Server and Wazuh SIEM Installation

This document explains how the **Wazuh SIEM server** was deployed in the SOC homelab.

The SIEM server runs on **Ubuntu Server** and is responsible for monitoring security events and detecting malicious activity.

---

## Virtual Machine Configuration

The Wazuh server runs inside a virtual machine managed by **KVM and Virt-Manager**.

Recommended configuration:

| Resource | Value |
|--------|------|
| OS | Ubuntu Server |
| RAM | 2-4 GB |
| CPU | 2 cores |
| Disk | 25 GB |
| Network | NAT (libvirt default network) |

---

## Install Ubuntu Server

1. Download the Ubuntu Server ISO from the official website.

2. Create a new virtual machine in Virt-Manager.

3. Select the ISO image and configure resources.

4. Install Ubuntu Server with default options.

During installation configure:

Hostname:

```
soc-lab
```

Create a user account.

Example:

```
username: sebastian_lab
```

After installation, log in to the server.

---

## Update the System

Update package lists and upgrade the system.

```bash
sudo apt update
sudo apt upgrade -y
```

---

## Install Wazuh SIEM

Download and execute the Wazuh installation script.

```bash
curl -sO https://packages.wazuh.com/4.7/wazuh-install.sh
sudo bash wazuh-install.sh -a
```

The **-a** option installs the full Wazuh stack:

- Wazuh Manager
- Wazuh Indexer
- Wazuh Dashboard

The installation process may take several minutes.

---

## Installation Output

At the end of the installation, the script generates credentials for the dashboard.

Example:

```
User: admin
Password: generated_password
```

These credentials are used to access the **Wazuh Dashboard**.

---

## Verify Services

Check that the Wazuh services are running.

```bash
sudo systemctl status wazuh-manager
```

```bash
sudo systemctl status wazuh-dashboard
```

---

## Access the Wazuh Dashboard

The dashboard runs on HTTPS.

Example:

```
https://localhost
```

Since the server runs inside a VM, access was configured using an **SSH tunnel**.

---

## SSH Tunnel to Access the Dashboard

From the host machine:

```bash
ssh -L 5601:localhost:443 sebastian_lab@192.168.122.X
```

Then open a browser on the host:

```
https://localhost:5601
```

Login using the Wazuh credentials.

---

## Password Recovery

If the dashboard credentials are lost, they can be regenerated.

Run:

```bash
sudo /usr/share/wazuh-indexer/plugins/opensearch-security/tools/wazuh-passwords-tool.sh --change-all
```

Restart services after changing credentials:

```bash
sudo systemctl restart wazuh-indexer
sudo systemctl restart wazuh-dashboard
```

---

## Network Verification

Verify communication between virtual machines.

From Kali Linux:

```bash
ping 192.168.122.X
```

Successful ping confirms that the attacker machine can reach the SIEM server.

---

## Result

The SOC homelab now includes a fully functional **Wazuh SIEM server** capable of detecting security events generated by other machines in the virtual network.
